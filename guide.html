Overview

Teal PMS is a full-featured project management tool that runs entirely in your browser — no server, no login, no subscription. It's inspired by Jira, Asana, and Zoho Projects, and supports epics, sprints, kanban boards, roadmaps, team workload, and rich reporting. You can import data from Zoho, Jira, Trello, and Asana, or start fresh with the built-in demo data.

💡 All data is stored in your browser's localStorage. Nothing leaves your device. Use the JSON backup feature regularly to preserve your work.

Getting Started

First Launch

On first launch, Teal PMS seeds three sample projects (ALPHA, BETA, GAMMA) with tickets, epics, and sprints so you can explore immediately. Head to the Dashboard to see the overview, or jump straight to Tickets or Board.

Switching Projects

Go to Projects and click Set Active on any project. All views (tickets, board, scrum, roadmap, reports) filter to the active project automatically.

Dark / Light Mode

Click the 🌓 toggle in the top right of any page to switch themes. Your preference is saved automatically.

Projects

Each project has a name, key (used as a prefix for ticket IDs, e.g. ALPHA-001), description, start date, and due date. Create as many projects as you need — each one is independent with its own tickets, epics, sprints, and milestones.

Click + New Project to create one. Click the card's menu to edit or delete. Click Set Active to make a project the current focus.

Tickets

Ticket Types

Teal PMS supports six ticket types: Task, Bug, Feature, Story, Improvement, and Epic. Each has its own color badge. Use types to categorize work — bugs for defects, features for new functionality, stories for user-facing requirements.

Statuses

Tickets flow through six statuses: Backlog → Todo → In Progress → Review → Done, plus Blocked for stalled items. Move tickets forward from the detail modal or by dragging on the Kanban board.

Priorities

Four priority levels: Critical (red), High (orange), Medium (teal), Low (blue). Set priority when creating or editing a ticket.

Filters & Search

Use the filter bar on the Tickets page to narrow by Status, Type, Priority, Epic, or Sprint. The search box matches ticket title and key. All filters combine — e.g. "Bug + Critical + In Progress" to find your top blockers.

Ticket Detail

Click any ticket to open the detail modal. Edit any field inline: title, description, status, type, priority, assignees, dates, labels, epic, sprint. Add timed comments. Log time spent with the + button in the Time Log section.

💡 Link a ticket to an epic and sprint to have it show up on the Roadmap and Scrum boards.

Kanban Board

The Board gives a visual drag-and-drop interface across six columns: Backlog, Todo, In Progress, Review, Done, and Blocked. Drag any card to change its status instantly.

Filter by Epic or Type using the dropdowns at the top. Each card shows ticket key, title, type badge, priority dot, and assignee avatars.

💡 Dropping a card into Done automatically closes it — no need to open the detail modal.

Scrum / Sprints

The Scrum view shows one sprint at a time with four columns: Todo, In Progress, Review, Done. Select the active sprint from the dropdown at the top.

Sprint Planning

The Backlog section at the bottom shows all tickets not yet assigned to a sprint. Click Add to Sprint on any backlog ticket to include it in the current sprint. Use this for sprint planning sessions.

Creating Sprints

Go to Projects → open a project → create sprints there, or use the sprint modal on the Scrum page (coming in v2).

Roadmap & Epics

The Roadmap shows all epics for the active project with progress bars, date ranges, and associated tickets. Each epic has a color, status (Todo / In Progress / Done), start date, and end date.

Click + Epic to create new epics. Link tickets to epics from the ticket detail modal's Epic dropdown. The roadmap updates progress in real time as tickets are moved to Done.

The Milestones panel below shows key project gates — deadlines, checkpoints, and deliverables. Create milestones from the Projects page.

Team & Workload

The Team page shows all team members with their role, department, and workload relative to the active project. Filter by department using the chips at the top.

Each member card shows how many tickets they're assigned to and how many are still open. The workload bar shows relative load compared to the rest of the team — useful for spotting overloaded members.

💡 Add team members from Settings (coming in v2). For now, assignee names are drawn from whoever you assign tickets to.

Reports

The Reports page provides analytical charts: Status doughnut, Priority doughnut, Tickets by Type bar chart, Assignee Workload horizontal bar, and Sprint Velocity tracker.

Scroll down for the Overdue Tickets table — a sorted list of all past-due open tickets with assignees and days overdue. This is the primary document for weekly project reviews.

Exporting

Click Export CSV to download a spreadsheet of all tickets for use in Excel. Click Backup JSON to export a full backup of all your Teal PMS data.

Importing Data

Go to Import in the sidebar to migrate data from other tools. Teal PMS supports five sources:

Zoho Projects — export your task list as CSV from Zoho Projects → Tasks → Export.

Jira — use Jira's Issues → Export → XML or use the Jira Cloud backup JSON.

Trello — open your board → Show Menu → More → Print and Export → Export as JSON.

Asana — open your project → ⋯ → Export/Print → CSV.

Teal PMS Backup — restore a previous JSON backup (replaces all data).

The import wizard auto-detects the format, shows a preview of tickets before committing, and lets you choose whether to import into an existing project or create a new one.

⚠ Restoring a full Teal PMS backup will overwrite all current data. Export a backup first if you want to preserve your current state.

Data & Backup

All Teal PMS data is stored in your browser's localStorage under the tpms2- key prefix. This is persistent across browser restarts but device-specific — data won't sync across computers automatically.

Regular Backups

Use Reports → Backup JSON regularly to save a full export. Store it on a shared drive or cloud folder so teammates can restore it on their machines.

Restoring a Backup

Go to Import → select Teal PMS Backup → upload your JSON file. Click Import to restore.

⚠ Clearing your browser's site data will erase all Teal PMS data. Always keep a recent JSON backup before clearing browser storage.
