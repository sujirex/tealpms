Overview

Teal PMS is a full-featured, browser-based project management tool — no server, no account, no subscription. It runs entirely in your browser using localStorage and works offline once loaded.

It is built for small-to-medium software and engineering teams and covers the full project lifecycle: ticket backlog, kanban board, sprint planning, gantt timeline, roadmap with epics, team workload, and analytical reports.

The live demo is at tealpms.pages.dev and comes pre-loaded with three sample projects (ALPHA, BETA, GAMMA) so you can explore every feature immediately.

All your data lives in the browser's localStorage. Nothing is sent to any server. Use Reports → Backup JSON regularly so you never lose work.

Getting Started

Demo Data

On first load, Teal PMS automatically seeds three demo projects with 35+ tickets, epics, sprints, milestones, and team members. You can explore everything immediately without any setup.

Switching Projects

Go to Projects in the sidebar and click Set Active on any project. All views — Tickets, Board, Sprints, Gantt, Roadmap, Reports — filter to the active project automatically. The active project key (e.g. ALPHA) is shown in the topbar.

Dark / Light Mode

Click Toggle Theme at the bottom of the sidebar (or the moon icon in the topbar) to switch between dark and light mode. Your preference is saved to localStorage automatically.

Clearing Demo Data

To start fresh with your own data, go to Import and upload a Teal PMS JSON backup, or clear your browser's localStorage for this site and reload. The demo seed will not re-appear once you have your own data loaded.

Projects

Each project has a name, short key (used as the ticket prefix, e.g. ALPHA-1), description, start date, due date, category, and owner. Create as many projects as you need — each is independent with its own tickets, epics, sprints, milestones, and team.

On the Projects page, click + New Project to create one. Click a project card to see its summary. Click Set Active to make a project the current working context for all other views.

The demo includes three projects: Project Alpha (software platform), Project Beta (marketing website), and Project Gamma (cloud infrastructure).

All tickets, charts, and reports automatically filter to whichever project is active. Switch freely — data for all projects is kept in localStorage simultaneously.

Tickets

Ticket Types

Six types are supported: Task, Bug, Feature, Story, Improvement, and Epic. Each has a distinct color badge. Use types to categorize work — Bugs for defects, Features for new functionality, Stories for user requirements, Improvements for enhancements.

Statuses

Tickets move through six statuses: Backlog → Todo → In Progress → Review → Done, plus Blocked for stalled items. You can change status from the ticket detail modal, the Kanban board (drag-and-drop), or the Sprint board.

Priorities

Four priority levels: Critical (red), High (orange), Medium (teal), and Low (grey). Overdue critical/high tickets show in red on the Tickets page.

Filters & Search

Use the filter bar to narrow by Status, Type, Priority, or Epic. The search box matches against ticket title and key. All filters combine — for example, "Bug + Critical + In Progress" shows only your highest-priority active defects.

Ticket Detail

Click any ticket row to open the full detail modal. From here you can edit every field: title, description, status, type, priority, assignees, start/due dates, labels, epic, percentage complete, and estimated hours. You can also add comments and log time spent.

Tickets with a due date in the past and status not Done show in red in the Due column and count toward the overdue badge on the sidebar.

Kanban Board

The Board page shows all tickets for the active project arranged in six columns: Backlog, Todo, In Progress, Review, Done, Blocked. Drag any card left or right to change its status instantly — no modal required.

Filter cards by Epic or Type using the dropdowns at the top. Each card shows the ticket key, title, type badge, priority, and assignee avatars. Click a card to open the full detail modal.

Dragging a card to the Done column marks it as completed. For blocked tickets, drag to the Blocked column to flag them for the team.

Sprints (Scrum)

The Sprints page shows a two-zone view: the active sprint board at the top and the ticket backlog at the bottom.

Sprint Board

Select any sprint from the dropdown. The board displays sprint tickets across four columns: Todo, In Progress, Review, Done. Progress bar and velocity stats are shown above the board.

Backlog & Sprint Planning

The backlog section shows all tickets not yet assigned to any sprint. Click Add to Sprint on any backlog ticket to include it in the selected sprint — this is your sprint planning interface.

Creating Sprints

Click + New Sprint on the Sprints page to create a new sprint with a name, goal, start date, and end date. Sprints belong to the active project.

The demo includes completed sprints (Sprint 1, Sprint 2) and an active sprint (Sprint 3) for Project Alpha, plus two sprints for Project Beta.

Gantt Chart

The Gantt page visualizes all tickets for the active project as horizontal time bars across a calendar timeline. It gives you an instant view of schedule, overlaps, and critical path.

Zoom Levels

Three zoom levels are available from the dropdown: Month (24 px/day, best for multi-month plans), Week (40 px/day, best for sprint detail), and Quarter (14 px/day, best for yearly overview).

Grouping

Group bars by Epic, Status, or view all tickets ungrouped. Tickets within each group are sorted by start date.

Today Line & Milestones

A teal vertical line marks today's date. Diamond markers on the timeline represent project milestones. Hover any bar or milestone to see a tooltip with ticket details, dates, and status.

Navigation

Click Today to scroll the timeline back to the current date. The timeline scrolls horizontally — use your mouse wheel or trackpad.

Roadmap & Epics

The Roadmap page shows all epics for the active project. Each epic displays its color, name, status badge, date range, progress bar (calculated from how many linked tickets are Done), and the list of associated tickets.

Click + New Epic to create an epic with a name, color, status, start date, and end date. Link tickets to an epic from the ticket detail modal's Epic dropdown — the roadmap progress bar updates automatically as tickets are completed.

Milestones are shown below the epics — key dates like releases, design sign-offs, or go-live events. Create milestones from the Projects page.

Epics are the parent grouping layer above tickets. Use epics for major themes like "User Authentication" or "Dashboard & Analytics" and add individual Feature/Task tickets inside each epic.

Team

The Team page shows all members defined in the active project's team roster. Filter by department using the chip filters at the top.

Each member card shows their name, role, department, avatar, and workload stats: total tickets assigned, open tickets, and a workload bar showing their relative load compared to the rest of the team. This makes it easy to spot overloaded members before assigning new work.

The demo includes 12 team members across Engineering, Design, QA, Management, Infrastructure, Analytics, and Marketing departments (User1 through User12 — generic names to protect confidentiality).

Assignees are set per-ticket in the ticket detail modal (comma-separated names). The Team page reads from the team roster in the project data and matches assignee names automatically.

Reports

The Reports page provides five analytical charts for the active project:

Ticket Status — doughnut chart showing the breakdown of tickets by status (Backlog, Todo, In Progress, Review, Done, Blocked).

Priority Distribution — doughnut chart showing Critical, High, Medium, Low ticket counts.

Tickets by Type — bar chart comparing Task, Bug, Feature, Story, and Improvement counts.

Assignee Workload — horizontal bar chart showing open ticket count per team member.

Sprint Velocity — bar chart comparing completed vs total tickets across sprints.

Below the charts, an Overdue Tickets table lists all past-due open tickets sorted by due date, with assignees and days overdue — useful for weekly project reviews.

Exporting

Click Export CSV to download a spreadsheet of all tickets. Click Backup JSON to export a complete backup of all Teal PMS data (projects, tickets, epics, sprints, milestones, team).

Importing Data

Go to Import in the sidebar to migrate data from other tools. The import wizard supports five sources:

Zoho Projects — export your task list as CSV from Zoho Projects → Tasks → Export.

Jira — use Jira's Issues → Export → XML (or the Jira Cloud backup JSON).

Trello — open your board → Show Menu → More → Print and Export → Export as JSON.

Asana — open your project → … → Export/Print → CSV.

Teal PMS Backup — restore a previous JSON backup exported from the Reports page.

The wizard auto-detects the format, previews the tickets before committing, and lets you choose whether to import into an existing project or create a new one.

Restoring a full Teal PMS backup will overwrite all current data. Export a backup first if you want to preserve your current state.

Data & Backup

All Teal PMS data is stored in your browser's localStorage under the tpms2- key prefix. Data persists across browser restarts but is device-specific — it will not sync to other computers automatically.

Regular Backups

Use Reports → Backup JSON to export a full snapshot. Store the file on a shared drive or cloud folder so team members can restore it on their machines, or so you have a recovery point before clearing the browser.

Restoring a Backup

Go to Import → Teal PMS Backup, upload your JSON file, and confirm the import. All projects, tickets, epics, sprints, milestones, and team data will be restored.

Resetting to Demo Data

Open your browser's DevTools → Application → Local Storage, find the tpms2-version key, and delete it. Reload the page — the demo seed data will reload automatically.

Clearing your browser's site data will erase all Teal PMS data permanently. Always export a backup before doing so.
